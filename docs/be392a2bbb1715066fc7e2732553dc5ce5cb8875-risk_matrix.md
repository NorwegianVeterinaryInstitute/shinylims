| Risk                                       | Specific failure mode in your setup                                                                          | Likelihood | Impact | Control                                                                                                                                        |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------ | ---------: | -----: | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Username path traversal                    | `username` contains `/` or `..` so constructed `/foo/bar/{username}/file.csv` escapes base                   |        Med |   High | Validate `username` with strict allowlist like `^[a-z_][a-z0-9_-]{0,31}$` then build path then `normpath` prefix check                         |
| Username mismatch vs real identity         | User types someone else’s username so you upload into their directory                                        |        Med |   High | Do not accept free username if you can avoid it; if you must then enforce server-side mapping from authenticated web identity to SAGA username |
| Credential leakage via exception/logging   | Unhandled exception or debug trace includes form dict with password or TOTP                                  |        Low |   High | Global logging filter that redacts sensitive keys; `debug=False`; no traceback echo in HTTP responses; disable core dumps                      |
| TOTP retry lockout                         | Users retry rapidly and SAGA locks account or rate-limits                                                    |        Med |   High | Rate limit per username and IP; no automatic retries; cooldown after N failures                                                                |
| Paramiko Transport misuse                  | Reuse across requests or threads corrupts channels even for 25 kB                                            |    Low-Med |   High | One Transport per request; no globals; close in `finally`                                                                                      |
| Overwrite of previous CSV                  | Same fixed filename overwrites prior upload silently                                                         |        Med |    Med | Unique filename (timestamp). We also explicit overwrite                                                                                        |
| Directory creation race or wrong ownership | Concurrent first uploads create dir with unintended perms or ownership                                       |    Low-Med |    Med | After mkdir verify ownership and mode explicitly; enforce fixed mode; We do not inherit from umask                                             |
